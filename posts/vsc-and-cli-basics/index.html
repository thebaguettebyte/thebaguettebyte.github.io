Visual Studio Code And Command Line Basics

Apr 3, 2025· baguettebye

A Gentleman’s Introduction to Visual Studio Code and the Command Line: Mastering the Tools of the Trade.

So, today’s learning curve brings us face-to-face with the enigmatic realm of IDEs… or is it IDE’s? No, no, let us avoid that grammatical minefield and simply stick to the singular: IDE. I won’t lie, I have dabbled in Visual Studio Code, or VSC, as the fashionable youth call it, mostly to wrangle this very blog into submission. It was a harrowing ordeal, given my complete lack of expertise, but I persevered! Of course, I leaned heavily on the wisdom of the theme’s discussion board and scoured Hugo’s official site for guidance. A humbling experience, to be sure.

First up! IDE? Huh…?

So, IDE stands for Integrated Development Environment, which is a rather posh way of saying, “Here’s a tool to help you type code”, I suppose. Sure, you could use Notepad if you fancied the challenge, or, in a truly adventurous spirit, you could scribble your code down with pen and paper, snap a photo, and ask an AI to transcribe it into code on your computer. The possibilities are endless, truly.

I have experimented with three different kinds of IDEs (though I refuse to count Notepad or the noble pen-and-paper method). As for the technical details of these programs, well, I shan’t pretend to be an expert. Frankly, I have no clue how half of it works, hah!

CLI

When I first encountered the term CLI, I had absolutely no idea what it meant, nooooo clue. Even when I learned it stood for Command Line Interface, I was none the wiser. I understood the words separately, but together? Swoosh! Right over my head.

After some digging, however, I’ve come to understand that a CLI is essentially a way to boss your computer around using text commands (and no, not the AI kind). You can create folders, move files, navigate your system, launch programs, and probably summon ancient digital spirits if you type the right incantation. Fascinating stuff!

The guide mentioned something about Git and the terminal, which was a delightful coincidence, turns out, I had already dabbled in this dark art when uploading my blog. And that’s when it finally clicked: a CLI is essentially your Command Prompt or PowerShell on Windows, or Terminal on macOS (which, I must confess, I had to look up).

Each CLI has its own collection of shortcuts and commands, a veritable treasure trove of wizardry that we shall uncover as we go.

For now, here are some of the commands I suspect I’ll be using most frequently:

Commands Explanation
cd.. Goes back one folder
mkdir Creates a folder
dir Shows what is inside said folder
